Instructions for ETF Imports:

  • Upload a Dhan ETF portfolio CSV exported from the Dhan portal
  • Expected columns: Scrip Name, Quantity, Avg. Buy Rate, Buy Value, LTP, Current Value, P&L, P&L%
  • The portfolio date from the file header (e.g. "For 24-06-2026") is used as the investment date unless you choose a custom date
  • The ETF issuer (e.g. HDFC, Kotak, ICICI Pru) is extracted as "Sub Type Name"
  • The ETF name without the issuer is set as "Sub Type Category"
  • If an investment with the same Platform, Sub Type Name, and Sub Type Category already exists, its value will be updated
  • New issuers or ETF categories not in existing data will be marked with 🆕

Instructions for Mutual Fund Imports:

  • Three CSV formats are supported:
    • HDFC Securities format: Columns like "Folio No", "Scheme Name", "Present Value"
    • MFPorfolioData format: Columns like "Fund", "Scheme", "Value At Cost"
    • Dhan App format: Columns like "Name", "Current Value"
  • The system will automatically detect the CSV format and extract the data accordingly
  • The system will automatically extract the AMC name as "Sub Type Name"
  • The scheme name (without AMC) will be set as "Sub Type Category"
  • If an investment with the same Platform, Sub Type Name, and Sub Type Category already exists, its value will be updated
  • Comprehensive AMC recognition including: Aditya Birla, Axis, HDFC, ICICI Prudential, SBI, Kotak, Nippon India, UTI, and many more
  • Properly handles complex scheme names like "HSBC Multi Cap Fund" (extracts HSBC as AMC and Multi Cap Fund as scheme)
  • Similar fund types are normalized (e.g., "Mid Cap Fund", "Midcap Fund", "Mid Cap Fund - Regular Plan" → "Mid Cap Fund")
  • New AMCs or scheme categories not in existing data will be marked with 🆕
